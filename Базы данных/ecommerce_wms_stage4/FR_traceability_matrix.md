# Таблица: требования из задания 1 → что мы сделали в SQL (этап 4)

Писали для себя чтобы не забыть что к чему относится. Предметка — склад, схема в postgres называется `ecommerce_wms`.  
Тексты требований смотрели в файле «Задание 1 БД (1).md» (он у нас в папке с конвертированными материалами).

Почему везде префикс `stage4_` — чтобы наши view/функции не пересеклись с тем что было на 3 этапе, так спокойнее.

| блок | что хотели по FR (очень кратко) | где смотреть в скрипте |
|------|--------------------------------|------------------------|
| R1 | заказ + строки, инциденты, корректировки, какая-то аналитика | сиды таблиц + `stage4_vw_order_detail` + `stage4_vw_incidents_returns` + функция `stage4_error_kpi` |
| R2 | операции, ячейки, «карточка» по времени | `stage4_vw_picking_ops` + функция `stage4_order_timeline` |
| R3 | задания на отбор и чуть статистики | те же сиды + `stage4_vw_picking_ops` + `stage4_picking_duration_stats` |
| R4 | сканы вес qc | сиды + `stage4_vw_picking_ops` (сканы) и `stage4_vw_quality_checks` |
| R4.3 фото | в нашей базе отдельной таблицы под фото нет | честно не делали, в отчёте написали почему |
| R5 | возврат + что-то для «выгрузки» | `return_order` в сидах + view + в конце sql просто большой запрос с несколькими `WITH` (отдельно view не делали, времени мало) |

Ещё отдельно для галочки по методичке:

- вложенные `select` / `exists` — внутри функций и в тестах внизу файла
- одна функция дергает другую — `stage4_order_health` зовёт `stage4_order_line_count`

Если что-то не сходится с формулировками препода — это норм, поправим по замечаниям.
