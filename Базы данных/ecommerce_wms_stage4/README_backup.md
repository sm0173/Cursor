# Как сделать backup после этапа 4

Мы сами на винде иногда ловили что `pg_dump` «не находится» — это значит postgres либо не ставили либо bin не в PATH. Тогда из pgAdmin делается backup через меню, тоже норм вариант.

## Что просят по заданию

Нужен файл дампа + наш dml sql. Старый `backup.sql` в папке «Базы данных» у нас вроде как custom формат (не открывается блокнотом нормально), восстанавливается через pg_restore. Для сдачи после 4 этапа логично снять **новый** дамп уже с нашими тестовыми данными.

**Лучше спросить у преподавателя** какой формат ему удобнее: custom или plain sql.

Примерно так в консоли (подставить свои host user dbname):

```
pg_dump -h localhost -U postgres -d ИМЯ_БАЗЫ -Fc -f backup_posle_etapa4.dump
```

plain вариант:

```
pg_dump -h localhost -U postgres -d ИМЯ_БАЗЫ -Fp --schema=ecommerce_wms -f backup_posle_etapa4.sql
```

Восстановление custom:

```
pg_restore -h localhost -U postgres -d НОВАЯ_БАЗА -c backup_posle_etapa4.dump
```

## В pgAdmin по шагам

1. восстановить базу с 3 этапа как раньше  
2. выполнить наш `ecommerce_wms_stage4_dml.sql`  
3. глянуть что внизу все `ok` = true  
4. правый клик по базе → backup, выбрать формат как договорились с преподом  

## P.S.

Если команды из readme не работают — не страшно, главное чтобы у кого-то из группы получилось и файл приложили к отчёту.
