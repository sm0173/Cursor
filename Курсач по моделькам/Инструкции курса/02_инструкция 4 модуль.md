# Инструкция — Модуль 04. Агенты и скиллы

## Часть 1. Зачем тебе AI-сотрудники

### Шаг 1. Принцип «Где задолбался — туда агента»

Не нужно придумывать, для чего бы применить AI. Вспомни задачи, которые:
- Ты делаешь каждый день и они бесят
- Занимают время, но не требуют гениальности
- Ты постоянно откладываешь, потому что «ну не хочется»

Вот реальные примеры:
- Задолбалась писать посты → AI-копирайтер
- Задолбалась искать файлы → AI-уборщик документов
- Задолбалась составлять отчёты → AI-аналитик
- Задолбалась отвечать на одни и те же вопросы → AI-менеджер

Запиши 3 задачи, на которые уходит больше всего времени. Первый агент будет для самой надоевшей.

**Результат:** У тебя есть конкретная задача для первого агента.

---

### Шаг 2. Что агент может и не может

Прежде чем создавать — пойми границы.

**AI может:**
- Рутина: обработка файлов, организация, шаблонные ответы
- Контент: посты, письма, описания, кейсы
- Анализ: отчёты, метрики, сравнения
- Документация: регламенты, инструкции, чеклисты

**AI не может:**
- Принимать стратегические решения за тебя
- Работать с людьми напрямую (звонки, переговоры)
- Делать физические действия
- Заменить глубокую экспертизу (юрист, врач)

**Зона роста** — то, чему агент учится со временем:
- Понимать твой стиль
- Адаптироваться под контекст
- Использовать успешные паттерны из прошлых задач

**Результат:** Ты понимаешь, что агент — это супер-стажёр с IQ 150, которому нужен контекст и чёткие задачи.

## Часть 2. Что такое .mdc и как это работает

### Шаг 3. Промпт vs Скилл

В **Модуле 01** мы создали `.cursorrules` — общий файл, который AI читает при каждом запросе. Но это один файл на весь проект.

**.mdc файлы** (Markdown Cursor) — это правила, которые подключаются по ситуации. Каждый .mdc — это отдельный «сотрудник» со своей ролью.

Разница:

| | Промпт | Скилл (.mdc) |
|---|---|---|
| Использование | Одноразовый | Многоразовый |
| Настройка | Каждый раз пишешь заново | Настроил один раз |
| Качество | Зависит от твоего настроения | Стабильно одинаковое |
| Улучшение | Никакого | Обновляешь файл — навсегда |

**Пример:**
- Промпт: «Напиши текст для сайта в профессиональном тоне, для предпринимателей, без воды, с call to action» — каждый раз писать заново
- Скилл: файл `копирайтер.mdc` — AI сам подключает его когда ты работаешь с текстовыми файлами

Где лежат: в папке `.cursor/rules/` внутри проекта.

**Результат:** Ты понимаешь, почему скиллы > промпты.

---

### Шаг 4. Структура .mdc файла

Каждый .mdc файл состоит из двух частей:

**Шапка** (между `---`):
```yaml
---
description: Когда AI подключит это правило
globs: ["**/*.md", "**/*.txt"]
alwaysApply: false
---
```

- `description` — описание для AI, когда подключить правило
- `globs` — для каких типов файлов правило работает
- `alwaysApply: false` — правило подключается по глобу, а не навязывается

**Тело** — сами инструкции для AI. Тут пишешь роль, задачи, правила, примеры.

**Результат:** Ты понимаешь структуру файла.

### Шаг 7. Создание файла правил (Шаг 3-6 из 7)

1. Открой настройки Cursor: **Cmd+Shift+J** (на Windows: Ctrl+Shift+J)
2. Перейди на вкладку **Rules**
3. Нажми **Add new rule** — Cursor создаст файл `.mdc` в папке `.cursor/rules/`
4. Назови файл `копирайтер` и заполни содержимое:

```markdown
---
description: Копирайтер для бизнес-текстов
globs: ["**/*.md", "**/*.txt", "**/*.html"]
alwaysApply: false
---

# КОПИРАЙТЕР

## РОЛЬ
Ты — копирайтер. Команда для вызова: `@копирайтер`

## КОНТЕКСТ
Папка агента: `агенты/КОПИРАЙТЕР/`

## ЗАДАЧИ

### Основные:
1. Посты для соцсетей
2. Тексты для сайта
3. Email-рассылки
4. Описания услуг

### НЕ входит:
- ❌ Анализ метрик (это → @аналитик)
- ❌ Дизайн (это → @дизайнер)
- ❌ Стратегия продвижения

## ПРАВИЛА
- Тон: профессиональный, но дружелюбный
- Абзацы: максимум 3 предложения
- Один текст = одна главная мысль
- Не использовать: «уникальный», «инновационный», «лидер рынка»
- Call to action в конце каждого текста

## ИНСТРУКЦИИ
При получении задачи:
1. Уточнить тему и формат (если не указаны)
2. Изучить примеры в `_КОНТЕКСТ/примеры/` (если есть)
3. Написать черновик
4. Проверить по правилам
5. Запросить оценку

## УТОЧНЯЮЩИЕ ВОПРОСЫ
При недостаточном контексте:
1. Задать 2-3 вопроса (не больше!)
2. Дождаться ответа
3. При необходимости — уточнить ещё

## САМООБУЧЕНИЕ
После каждой задачи:
1. Запросить оценку (✅ Отлично / ⚠️ Нужны правки / ❌ Не то)
2. При «Отлично» → сохранить в `_КОНТЕКСТ/примеры/`
3. Логировать в `_ЛОГИ/`

## ПРИМЕРЫ

### Пример: Продающий пост
**Запрос:** «Напиши пост о запуске курса по AI»
**Результат:**
Через 7 дней ты будешь работать по-другому.

Не «как стать программистом». А как перестать тратить 4 часа на то, что AI делает за 15 минут.

Курс «Cursor: от нуля до автоматизации»:
• 8 модулей практики
• 0 воды
• Результат с первого урока

→ Записаться: [ссылка]
```

3. Создай папку для агента:
```
mkdir -p агенты/КОПИРАЙТЕР/_КОНТЕКСТ/примеры
mkdir -p агенты/КОПИРАЙТЕР/_ЛОГИ
```

4. Создай `агенты/КОПИРАЙТЕР/README.md`:
```markdown
# КОПИРАЙТЕР

## Назначение
Создание текстового контента для бизнеса.

## Команда
`@копирайтер`

## Задачи
1. Посты для соцсетей
2. Тексты для сайта
3. Email-рассылки

## Как использовать
`@копирайтер напиши пост о [тема]`
```

**Результат:** Первый агент создан — и файл правил, и папка контекста.

### Шаг 8. Тестирование (Шаг 7 из 7)

Протестируй агента на трёх уровнях:

**Простая задача** — проверяем базу:
```
@копирайтер напиши короткий пост о нашем продукте
```

**Типичная задача** — реальная ситуация:
```
@копирайтер напиши пост для Telegram: тема — как AI экономит 4 часа в день, формат — 100-150 слов, стиль — дружелюбный
```

**Сложная задача** — проверяем границы:
```
@копирайтер напиши серию из 3 постов для запуска курса: анонс, напоминание, последний шанс
```

После каждого теста оцени:
- AI соблюдает тон из правил?
- Формат ответа правильный?
- Нет запрещённых слов?

Если что-то не так — обнови файл `.mdc` и протестируй снова.

**Результат:** Агент проверен и работает.

## Часть 4. Выбор моделей

### Шаг 9. Какие модели бывают

В Cursor можно переключать модели AI. Внизу окна Composer есть выпадающий список.

Модели делятся на 4 категории по задачам:

**Логика и анализ** — найти ошибку, проанализировать отчёт, построить стратегию:
- Claude Sonnet — отличная логика, хороший русский
- Gemini 2.5 Pro — огромный контекст (1M токенов), отличная структура
- GPT-4o — стабильная классика

**Креатив и текст** — написать пост, письмо, слоган:
- Claude Sonnet — лучший «человечный» слог, без клише
- Gemini Flash — хорошо понимает русский сленг и нюансы

**Код и техника** — написать скрипт, исправить баг:
- Claude Sonnet — эталон для Cursor
- GPT-4o — стабильная база

**Черновики и объём** — обработать 100 файлов, перевести текст:
- DeepSeek — мощность за копейки
- Gemini Flash — молниеносно и почти бесплатно

### Шаг 10. Как переключить модель

1. Открой Composer (Cmd+I / Ctrl+I) или Chat (Cmd+L / Ctrl+L)
2. Внизу окна — название текущей модели
3. Кликни → выпадающий список
4. Выбери другую модель

Главное правило: **не влюбляйся в модель**. Они обновляются каждый месяц. То, что лучше сегодня — завтра может быть вторым.

Где смотреть актуальные рейтинги:
- **openrouter.ai/rankings** — рейтинг моделей по качеству, скорости и цене. Самый актуальный и удобный

**Результат:** Ты знаешь, какие модели для чего, и умеешь переключать.

---

### Шаг 11. Тройной прогон

Для важных задач (текст для клиента, стратегия) используй Тройной прогон:

1. Выполни задачу в Claude Sonnet
2. Переключи модель → выполни ту же задачу в GPT-4o
3. Сравни результаты
4. Возьми лучшее из обоих (или попроси третью модель объединить)

Зачем: разные модели «думают» по-разному. Claude лучше пишет тексты, GPT точнее с фактами, Gemini лучше работает с длинными документами.

Когда использовать: только для важных задач. Для ежедневных мелочей — одной модели достаточно.

**Результат:** Ты попробовал одну задачу на двух моделях и видишь разницу.

## Часть 5. Система самообучения

### Шаг 12. lessons.md — память агента

AI не помнит прошлые разговоры. Файл `lessons.md` — это его долгосрочная память.

1. Создай файл `lessons.md` в корне проекта
2. Формат записи:

```markdown
## 2026-04-03

### Проблема
Копирайтер использовал слово «уникальный» в тексте, хотя в правилах запрещено.

### Решение
Добавила в .mdc: «Перед финальным ответом проверь текст на запрещённые слова.»

### Результат
Больше не использует запрещённые слова.
```

3. Добавь в `.cursorrules` строку:
```
Перед каждым ответом проверяй lessons.md — не повторяй ошибки из этого файла.
```

**Результат:** У тебя есть система улучшения агентов.

---

### Шаг 13. Самообучение через контекст

Это главное отличие живой системы от статичного промпта.

Каждый агент собирает свои успешные кейсы:

```
агенты/КОПИРАЙТЕР/
├── _КОНТЕКСТ/
│   └── примеры/
│       ├── пост_запуск_курса.md    ← оценка: отлично
│       └── email_клиенту.md        ← оценка: отлично
├── _ЛОГИ/
│   └── 2026-04-03.md
└── README.md
```

Как это работает:
1. Ты даёшь задачу → агент выполняет
2. Ты оцениваешь: ✅ Отлично / ⚠️ Нужны правки / ❌ Не то
3. При «Отлично» — агент сохраняет запрос + результат в `_КОНТЕКСТ/примеры/`
4. В следующий раз агент **сначала смотрит примеры**, потом делает

Чем больше используешь — тем лучше работает. Как стажёр, который учится на своих ошибках.

**Результат:** Агент улучшается с каждым запросом.

## Часть 6. Масштабирование

### Шаг 14. Второй агент — аналитик

Создай второго агента — для другой задачи.

1. Открой **Cmd+Shift+J** → **Rules** → **Add new rule**, назови `аналитик`:

```markdown
---
description: Аналитик бизнес-данных
globs: ["**/*.csv", "**/*.xlsx", "**/*.json"]
alwaysApply: false
---

# АНАЛИТИК

## РОЛЬ
Ты — бизнес-аналитик. Команда: `@аналитик`

## ЗАДАЧИ
### Основные:
1. Анализ данных и метрик
2. Создание отчётов
3. Сравнение показателей

### НЕ входит:
- ❌ Написание текстов (это → @копирайтер)
- ❌ Прогнозирование рынка

## ФОРМАТ ОТВЕТА
1. Главный вывод (1-2 предложения)
2. Ключевые цифры (3-5 пунктов)
3. Что делать дальше (1-2 рекомендации)

## ПРАВИЛА
- Считай в конкретных числах, не в процентах без базы
- Если данных мало — скажи прямо, не додумывай
- Сравнивай с прошлым периодом если есть данные
- Формат чисел: 1 234 567 (с пробелами)
```

2. Создай тестовый файл `продажи.csv`:
```
Месяц,Выручка,Расходы
Январь,500000,320000
Февраль,620000,310000
Март,580000,340000
```

3. В Chat:
```
@File продажи.csv Проанализируй данные
```

**Результат:** Второй агент работает. AI отвечает в формате «вывод → цифры → рекомендации».

### Шаг 15. Мульти-агентная работа

Когда агентов несколько — они могут передавать задачи друг другу.

**Простой способ (один Cursor):**
- Несколько .mdc файлов с разными ролями
- AI подхватывает нужный .mdc по типу файла автоматически
- Ты указываешь через `@` кого вызвать

**Продвинутый способ (несколько окон):**
- `git worktree add` — несколько копий проекта
- Каждое окно Cursor — отдельный агент
- Координация через общую папку

Реальный кейс в `материалы/кейс_3_агента_claude_code.md` — как один разработчик за 4 недели собрал SaaS, используя 3 агента параллельно.

**Результат:** Ты знаешь два способа мульти-агентной работы.

### Шаг 17. Hooks — автоматические проверки

Hooks — действия, которые выполняются автоматически после каждого ответа AI.

Пример: после того как агент изменил файл — автоматически отформатировать код.

Создай `.cursor/hooks.json`:

```json
{
  "post-apply": [
    {
      "command": "prettier --write {files}",
      "description": "Форматирую код"
    }
  ]
}
```

Подробнее — в `материалы/hooks_автопроверки.md`.

**Результат:** Ты понимаешь что такое hooks и как они автоматизируют проверку.
