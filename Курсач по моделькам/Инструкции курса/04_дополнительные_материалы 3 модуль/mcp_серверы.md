# MCP-серверы — подключаем внешние сервисы к Cursor

## Что такое MCP

**MCP (Model Context Protocol)** — стандартный протокол, через который AI-редакторы (Cursor, Claude Code, Windsurf) подключаются к внешним сервисам. Вместо того чтобы вручную копировать данные из Gmail или Notion в чат — AI сам обращается к ним напрямую.

Представь: вместо «скопируй письмо из почты и вставь в Cursor» — ты пишешь `Проверь мою почту и найди письма от клиентов за сегодня`, и AI сам лезет в Gmail.

## Как подключить

В Cursor: **Settings → MCP** (или команда `/mcp` в чате).

Формат конфигурации (`.cursor/mcp.json`):
```json
{
  "mcpServers": {
    "название-сервера": {
      "command": "npx",
      "args": ["-y", "название-пакета"],
      "env": {
        "API_KEY": "твой-ключ"
      }
    }
  }
}
```

## Полезные MCP-серверы для бизнеса

### 📧 Gmail
Читать, искать и отправлять письма прямо из Cursor.
- «Найди все письма от клиента Иванов за эту неделю»
- «Напиши ответ на последнее письмо»

### 📅 Google Calendar
Просмотр и создание событий.
- «Какие встречи у меня завтра?»
- «Добавь встречу с клиентом на пятницу 15:00»

### 🎨 Figma
Доступ к дизайн-макетам.
- «Возьми макет главной страницы из Figma и сверстай его»

### 💬 Telegram
Чтение и отправка сообщений.
- «Отправь отчёт в рабочий чат»

### 🌐 Browser
AI управляет браузером — открывает страницы, кликает, заполняет формы.
- «Открой сайт конкурента и сделай скриншот главной»

### 📝 Notion
Чтение и редактирование баз данных в Notion.
- «Добавь нового клиента в CRM в Notion»
- «Покажи все задачи со статусом "В работе"»

## Важно

- MCP-серверы работают локально на твоём компьютере
- API-ключи хранятся у тебя, не отправляются третьим лицам
- Не все серверы стабильны — проверяй перед использованием в продакшне
- Некоторые серверы требуют Node.js (npx) или Python (uvx)

## Практика

1. Выбери один MCP-сервер (начни с Browser или Notion)
2. Установи и подключи через Settings → MCP
3. Проверь что работает — задай вопрос, связанный с подключённым сервисом
