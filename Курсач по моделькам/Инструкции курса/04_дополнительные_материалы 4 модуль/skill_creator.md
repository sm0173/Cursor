# Skill Creator — создание скиллов для AI-агентов

> Скилл — это модульный пакет, который превращает AI из универсального помощника
> в специалиста в конкретной области. По сути — «инструкция» для нейронки.

## Что такое скилл

Скилл — это папка с файлами:

```
мой-скилл/
├── SKILL.md              ← главный файл (обязательный)
├── scripts/              ← скрипты (Python/Bash)
├── references/           ← справочные материалы
└── assets/               ← шаблоны, картинки, файлы для вывода
```

**SKILL.md** — единственный обязательный файл. В нём:
- Метаданные (имя + описание) — AI читает их чтобы понять, когда использовать скилл
- Инструкции — что делать, как делать, какие файлы использовать

## Как создать скилл

### Вариант 1: Попросить нейронку (самый простой)

Просто опиши что хочешь. Например:

```
Создай скилл для проверки орфографии в markdown-файлах.
Он должен:
- Находить ошибки в русском тексте
- Предлагать исправления
- Проверять заголовки на единообразие
```

Нейронка сама создаст структуру, напишет SKILL.md и скрипты.

### Вариант 2: Инструменты для создания

- **skill-creator** (Anthropic) — встроенный скилл для Claude Code / OpenClaw
  GitHub: https://github.com/anthropics/skills/tree/main/skills/skill-creator

- **Agent Skill Creator** (универсальный) — работает с любыми инструментами
  GitHub: https://github.com/FrancyJGLisboa/agent-skill-creator

### Вариант 3: Вручную

1. Создай папку с именем скилла (lowercase, через дефис)
2. Создай `SKILL.md` с frontmatter:

```yaml
---
name: мой-скилл
description: Что делает скилл и когда его использовать. Пиши подробно — AI читает это чтобы понять, нужен ли скилл для текущей задачи.
---
```

3. Напиши инструкции в теле SKILL.md
4. Добавь скрипты/справки/ассеты по необходимости

## Структура SKILL.md

```markdown
---
name: проверка-текстов
description: Проверка и редактирование текстов на русском языке. Использовать когда пользователь просит проверить орфографию, стилистику, или отредактировать текст.
---

# Проверка текстов

## Быстрый старт
1. Прочитай файл
2. Проверь орфографию
3. Проверь стилистику
4. Предложи исправления

## Правила проверки
- Не менять авторский стиль
- Исправлять только явные ошибки
- Для спорных случаев — предлагать варианты

## Скрипты
- `scripts/check_spelling.py` — автопроверка орфографии
```

## Принципы хороших скиллов

### Краткость
AI и так умный. Добавляй только то, что он не может знать сам:
- Специфику твоего проекта
- Правила твоей компании
- Конкретные шаблоны и форматы

### Степень свободы
- **Много свободы** — когда подходов много (написание текстов, дизайн)
- **Средняя** — когда есть предпочтительный паттерн (формат отчёта)
- **Мало свободы** — когда важна точность (работа с API, деплой)

### Прогрессивная загрузка
Не пихай всё в SKILL.md. Раздели:
1. **Метаданные** (имя + описание) — AI видит всегда (~100 слов)
2. **SKILL.md** — загружается когда скилл нужен (до 500 строк)
3. **references/** — загружается по необходимости (без ограничений)

## Примеры скиллов

### Скилл для ведения соцсетей
```
smm-помощник/
├── SKILL.md              ← правила постинга, tone of voice
├── references/
│   ├── instagram.md      ← форматы, хештеги, время постинга
│   ├── telegram.md       ← форматы каналов, кнопки
│   └── тон-бренда.md     ← примеры текстов в стиле бренда
└── assets/
    └── шаблон-поста.md   ← шаблон структуры поста
```

### Скилл для учёбы
```
егэ-физика/
├── SKILL.md              ← как решать задачи, формат ответов
├── scripts/
│   └── генератор_задач.py
└── references/
    ├── формулы.md        ← все формулы по темам
    ├── ошибки.md         ← типичные ошибки учеников
    └── критерии.md       ← критерии оценивания ЕГЭ
```

### Скилл для фриланса
```
фриланс-ассистент/
├── SKILL.md              ← как отвечать на заказы, считать цены
├── references/
│   ├── прайс.md          ← прайс-лист услуг
│   ├── шаблоны-ответов.md ← готовые ответы заказчикам
│   └── портфолио.md      ← описание работ
└── assets/
    └── шаблон-кп.md      ← шаблон коммерческого предложения
```

## Как установить скилл

### В Cursor
Положи папку скилла в `.cursor/skills/` в корне проекта.

### В OpenClaw
```bash
openclaw skill install путь/к/скиллу
```

### В Claude Code
Положи в `~/.claude/skills/` или укажи путь в конфиге.

## Где найти готовые скиллы

- **ClawhHub**: https://clawhub.com — маркетплейс скиллов для OpenClaw
- **Anthropic Skills**: https://github.com/anthropics/skills — официальные скиллы
- **Сообщество**: https://discord.com/invite/clawd

## Главное

Скиллы — это не сложно. Ты просто описываешь что нужно, а нейронка делает остальное. Начни с простого скилла (проверка текста, генерация постов) и усложняй по мере необходимости.
