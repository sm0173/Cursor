# Прога — материалы курса и контекст для ИИ

Здесь лежат учебные материалы по **Python** и **интеллектуальному анализу данных (ИАД)** в виде текстовых выгрузок, плюс скрипт конвертации из исходников. Этот файл — **единая точка входа**: правила ниже задают, **какой контекст использовать** при вопросах и при написании кода в этом проекте.

## Эталонная структура зоны

Ориентир — как в шаблонах курса (отдельно **документация**, отдельно **скрипты**, отдельно **материалы**):

```
Прога/
├── README.md                 ← вы здесь: правила и запуск
├── docs/                     ← навигация и каталог файлов
│   ├── README.md
│   └── INDEX.md
├── scripts/                  ← конвертация и зависимости Python
│   ├── convert_materials.py
│   └── requirements-materials.txt
├── 01_Оригиналы материалов/  ← исходники (не основной контекст для ИИ)
├── 02_Материалы по ИАДу/     ← выгрузки Sem …
└── 03_Материалы по питону/   ← выгрузки Seminar…
```

Префиксы **`01_` … `03_`** у папок с материалами задают **понятный порядок** в проводнике. Полный перечень имён файлов по папкам — [`docs/INDEX.md`](docs/INDEX.md).

## Назначение папок

| Папка / зона | Содержимое |

## Обязательные правила для ИИ (и для тебя при работе в Cursor)

1. **Не использовать как учебный контекст** содержимое папки **`Прога/01_Оригиналы материалов`**: при ответах на вопросы по курсу, при разборе тем и **при написании кода** опирайся не на ноутбуки, а на выгрузки (см. п. 2). Исключение — только задачи вроде «запусти конвертацию», «исправь скрипт», «добавь файл в оригиналы».
2. **Основной контекст для анализа и кода** — файлы **`Прога/02_Материалы по ИАДу`** и **`Прога/03_Материалы по питону`** (в первую очередь **`.txt`**). Имена `Sem …` = ИАД, `Seminar…` = Python.

Эти два пункта **не отменяют** остальные правила репозитория (`.cursorrules`, `tasks/`), а уточняют, **откуда брать знания** в зоне `Прога/`.

---

## Дополнительные правила (оптимизация контекста)

- **База для кода:** стиль библиотек, приёмы и формулировки задач брать из релевантных **`.txt`**. Если файлы большие — читать **фрагменты**, а не целиком без необходимости.
- **Согласованность с курсом:** при генерации кода стараться **совпадать с подходом из материалов**, если пользователь не просит иной стандарт.
- **Точность:** в выгрузках могут быть артефакты конвертации (например, длинный base64). Для смысла и кода ориентироваться на **текст и блоки кода**.
- **Актуальность:** после правок в оригиналах перезапусти конвертацию (команды ниже).

---

## Как пользоваться в Cursor

- **`@Прога/docs/INDEX.md`** — полный каталог файлов по всем папкам зоны.
- **`@Прога/README.md`** — правила контекста; затем **`@` конкретный `.txt`** из `02_…` или `03_…`.

---

## Обновление текстовых материалов

Из корня репозитория:

```bash
pip install -r "Прога/scripts/requirements-materials.txt"
python "Прога/scripts/convert_materials.py"
```

Из папки `Прога` (если терминал уже в ней):

```bash
pip install -r "scripts/requirements-materials.txt"
python "scripts/convert_materials.py"
```

Опции: `--ipynb-only`, `--pdf-only` — см. docstring в `scripts/convert_materials.py`.

---

**Цель:** ответы и код опираются на **`.txt`** в `02_` и `03_`, без опоры на ноутбуки из **`01_Оригиналы материалов`** как на основной источник знаний.
